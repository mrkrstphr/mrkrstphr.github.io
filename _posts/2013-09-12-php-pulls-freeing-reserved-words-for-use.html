'a:1:{i:0;s:65:"PHP Pulls: Freeing Reserved Words for Use? - http://goo.gl/8dHg2t";}' --- A developer named bwoebi has submitted a pull request to PHP that would allow - in certain circumstances - previously reserved words to be used as class and method names. His example shows a class named List with a method of default(). This would be pretty sweet and has been, at times, something that has frustrated me when I've wanted a method called something like list, but had to go with getList() instead due to list being a reserved word. The only problem with this implementation is that it leads to a lot of ambiguity and unintended consequences, such as writing a function named if and then not knowing if the one should call the conditional if or the function. I'm very interested to see where this goes. I'd definitely support it in the case of Class and Method names.
