--- layout: post title: Introducing Clover Dump categories: - Development - Tools tags: - clover - continuous integration - PHP - phpunit - Tools status: draft type: post published: false meta: _edit_last: '2' _jd_tweet_this: 'yes' --- I wrote a tiny little tool that simply takes a clover.xml file and dumps out the coverage information to the console. Why is this useful? I use it for Continuous Integration servers, like Travis CI so that I have a visual representation of my code coverage. Add it to your project's dependencies with Composer: 
require: {
    "clover/dump": "1.*"
}
Using it is simple: 
$ vendor/bin/clover-dump build/logs/clover.xml
The output will be similar to: 
Clover Code Coverage Report:

 - 100.00% PhpGedcom\Gedcom
 -   0.00% PhpGedcom\Record
 -   0.00% PhpGedcom\Record\Addr
 -  50.00% PhpGedcom\Record\Caln
 -   0.00% PhpGedcom\Record\Chan
 -  37.93% PhpGedcom\Record\Fam
 -  95.83% PhpGedcom\Record\Head
 -  38.26% PhpGedcom\Record\Indi
 -  61.54% PhpGedcom\Record\Note
Code Coverage: 55.19%
It's that simple. Suggestions, comments, bugs, enhancements? Hit up the GitHub page.
