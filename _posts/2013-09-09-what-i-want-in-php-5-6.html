--- layout: post title: What I Want in PHP 5.6 categories: - Development tags: - development - Ideas - PHP status: draft type: post published: false meta: _edit_last: '2' _jd_tweet_this: 'yes' --- PHP 5.5 has been available for a few months now, and I'm already getting excited at what might be released in PHP 5.6. Versions 5.0 through 5.5 have each iteratively brought some very exciting features, and I'd love to see that process continue. Already accepted is an RFC that allows for importing namespaced functions, just like you can with classes. That one doesn't excite me too much as I don't really use class-less functions in my day-to-day coding. There are quite a few things I'd love to see someday:

1. Type-Hinting for Scalar Values

PHP 5 introduced type hinting for class and array (and later, callable) arguments to functions and methods, which is fantastic. This feature allows you to ensure that the data being passed to your function is exactly what you expected it to be. But this same feature doesn't yet apply to scalar values, such as string, int, float, etc. This feature has several caveats with implementing as PHP is a loosely typed language. All incoming POST data is automatically a string, so if you type hinted for an int, your code might blow up. But PHP already does a good job at handing this, like whenever you do math on a string and it automatically gets converted to an integer or float. I don't think it'd be a big deal to extend this same automatic conversion to function/method type hinting as well. There have been several RFCs discussing these features, and, unfortunately, none of them have ever really gained traction.

3. Named Parameters

Named parameters is a sweet feature that allows you to specify the parameters of a function or method in any order you want by naming them with the value. This also allows you to skip optional parameters, rather than supplying the default. Consider a function that takes a handful of arguments: 
public function update($customerId, $orderNumber, $shipDate = null, $total = 0, $comments = '')
{
    // ...
}
A recently submitted RFC has brought this feature back up again.

4. ReflectionAnnotation Support

PHP has a great set of Reflection classes that allow you to figure out just about anything you want about a class, method, member variable, etc, but it doesn't have built in support to parse annotations/docblocks. It only allows you to get the annotations as a blob of text. There are several times when these dockblock annotations can be used to help drive implementation or discover information about a class, especially since PHP isn't a strongly typed language with type-hinting on method calls (see #1) above. Several people, including myself, have written their own library to parse these, but it'd be nice if PHP built in the support natively to compliment the rest of the built-in Refelection classes. Again, an RFC is already available for this requested feature.
